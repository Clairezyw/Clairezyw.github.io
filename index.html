shouldComponentUpdate

在了解这两个东西之前，先要明确一下render的渲染机制。
首先，render会回传想要渲染的内容，react会把渲染的内容转换为虚拟节点，最后经过react的diff算法进行比较，把更新的内容更新的真实的节点，从而完成渲染。
这样的话，减少渲染的次数就可以提升react性能。
这时就会想到使用shouldComponentUpdate，让它返回false，这样的话就不会调用render函数。

js判断scroll滚动方向

Veröffentlicht am 2018-04-16 | in javascript . 滚动方向 
 function controller() {
        window.onscroll = function () {
            getDirection();
            if (direction === "down") {
                console.log("down");
                //do something
            }
            if (direction === "up") {
                console.log("up");
                 //do something
            }
            if (direction === 'right') {
              console.log("right");
                 //do something
            }
            if (direction === "left"){
                console.log("left");
                 //do something
            }
        }
    }

    var a = {x: 'undefined', y: 'undefined'}, direction;

    function getDirection() {
        if (typeof a.x === 'undefined') {
            a.x = window.pageXOffset;
        }
        if (typeof a.y === 'undefined') {
            a.y = window.pageYOffset;
        }
        var diffx = a.x - document.documentElement.scrollTop;
        var diffy = a.y - document.documentElement.scrollTop;
        if (diffy <= 0) {
            direction = "down";
        } else if (diffy > 0) {
            direction = "up";
        }
        else if (diffx <= 0) {
            direction = 'right';
        }
        else {
            direction = 'left';
        }
        a.x = window.pageXOffset;
        a.y = window.pageYOffset;
    }

    document.onscroll = controller;

关于作用域

Veröffentlicht am 2018-03-22 | in JavaScript . 作用域

理解作用域

1、作用域其实就是一套规则，用于确定何处以及如何查找变量，如果要对变量赋值，就会执行LHS查找。如果是查找变量的值就会执行RHS查找。（RHS是指变量在右侧，LHS是指变量在左侧）。
2、LHS和RHS查找都会在当前作用域内进行查找，如果当前作用域没有目标变量他们就会向上级作用域内进行查找，最后抵达全局作用域，无论有没有找到都会停止。
3、如果RHS查找一个未声明的变量就会抛出一个ReferenceError的异常，但是在非严格模式下LHS查询则会创建一个全局变量，严格模式下也会抛出一个ReferenError的异常
