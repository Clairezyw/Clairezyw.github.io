promise

promise的作用就是异步处理，目的就是解决es5异步函数结构复杂、不清晰的问题。看下面简单的例子

//创建用户类
 class User{
 constructor(name,password){
    this.name = name;
    this.password = password;
 }
 //检测用户名是否正确
  testName(cb){
  let name =this.name;
    return new Promise(function(resolve,reject){
        if (name === 'stalary'){
         resolve('success');
    }
          else{
            reject('error');
       }
  });
}

//检测密码是否正确
testPassword(cb){
   let password =this.password;
   return new Promise(function(resolve,reject){
   if(password==='123'){
        resolve('success');
   }   else{
             reject('error')
      }
  });
 } 
}
var user =new User('stalary','123');
user.testName()
    .then(function(res){
       return user.testPassword;
    })
    .then(function(res){
       console.log('验证成功！')；
    })
    .catch(function(error){
      console.log('验证失败！')；
    })；

1、在上述例题中，testName()方法中返回了一个promise对象，这个对象只有一个function参数，并且函数中包含两个参数，resolve和reject，当函数正常执行时，执行resolve的操作。当出现异常时，执行reject操作。
2、then（）方法
在创建user实例时，调用testName()方法，这时我们会发现代码中有then（）方法，该方法是promise的方法，用于正常返回一个promise对象时执行的操作，在这里当用户名为stalary时，成功返回了一个promise对象，然后在控制台输出验证成功
3、catch（）方法用于捕获异常，当用户名不正确或者密码不正确时，会输出验证失败

promise的异常处理

还是上述例子，我们来看一下不同情况下的异常处理

 }

var user =new User('claire','123');
user.testName()
    .then(function(res){
    //第一种异常处理
    throw Error('first error');
       return user.testPassword;
    }//第二种异常处理
    ,function(err){
       console.log('second error');
    }
    )
    .then(function(res){
       console.log('验证成功！')；
    })
    //第三种异常处理
    .catch(function(error){
      console.log('验证失败！')；
    })；

如果我们把用户名改成claire，想一想会报出哪种错误呢？
先说一下第二种异常处理，我们上面讲到promise对象的函数里面有两个参数，当正常执行时，执行的是resolve，当抛出异常时执行的时reject，

then（function fun1(res){
            ...
},function fun2(err){
            ...
   }
）

当正常执行时与之相对应的是then方法中的第一个函数，抛出异常时对应第二个函数，这时候就有疑问了，catch也是捕获异常的方法，当第二个函数和catch同时存在时，会执行fun2，不会执行catch，如果没有第二个函数，那么会执行catch进行异常处理。
这个问题最终的结果是会输出second error，原因是因为我们把用户名更改以后，用户名是错误的，相应的会执行reject的操作，就是对应的fun2的操作。

promise的方法

如果我们要验证多个用户时，重复输入是件很麻烦的事，promise中的all（）方法解决了这个问题，该方法将所有的promise结果放在一个数组中，并返回一个promise对象

var user =Promise.all([new User('stalary','123')],[new User('claire','123')]);
user.then(function(res){
  console.log(res);//[ User { name: 'stalary', password: '123' } ]
 // 验证失败

})

对于stalary用户来说验证成功，并输出用户的信息，对于其他用户来说验证失败
除此之外，还有race（）方法，该方法和all（）方法类似，只不过返回的不是所有的结果，而是最先完成后的一个结果
