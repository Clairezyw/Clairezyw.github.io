set

es6提供了新的数据结构set，该结构有点类似于数组，但其值是不可重复的。
下面来看一下常用的方法：
1、add（） 添加一个元素
2、delete（）删除一个元素
3、has（） 判断是否包含某个元素
4、clear（）清空所有的元素

let arr = ['name:claire','age:18','age:18'];
let st  = new Set(arr);
//age：18　重复，只保留一个age
console.log(Array.from(st));//name:claire age:18
//has方法
console.log(st.has('name:claire'));//true
　//delete
st.delete("age:18");
console.log(st);//name:claire
st.add(22);
console.log(st)//name:claire age:18 22
//clear()
st.clear();//st为空

DOM2级对于dom节点的遍历提供了两种深度优先遍历的类型：NodeIterator和TreeWalker,下面来介绍这两种类型的使用。
1、NodeIterator
该类型通过document.createNodeIterator()方法创建新实例，该方法接受四个参数：

<1>root：表示开始遍历的起点

<2>whatToShow：表示访问哪些节点的数字代码，以常量的形式作为参数。
常见的参数有：
NodeFilter.SHOW_ALL:显示所有类型的节点
NodeFilter.SHOW_ELEMNT:显示元素节点
NodeFilter.SHOW_ATTRIBUTE：显示属性节点
NodeFilter.SHOW_TEXT：显示文本节点

<3>filter:表示应该接受某个特定节点还是应该拒绝某个特定节点的函数

<4>entityRefernceExpasion:布尔值，表示是否要扩展实体，在html中不起作用，因此为false

javascript闭包问题

Veröffentlicht am 2017-12-05

javascript支持闭包功能，函数可以引用作用域外的变量，但在使用闭包时存在一定的陷阱，就像下面的这个例子：

   var arr = [];    
   for(var i = 0;i<3;i++){
       arr.push(i); 
   }
   for (var j = 0;j<arr.length;j++){
   alert(arr[j]);}

输出结果为：3 3 3
并不是我们想要的结果：0 1 2
这个问题的原因是javascript虽然有局部变量，
但局部变量只有函数的作用域没有块作用域，因此无论在哪个位置定义变量，变量的作用域都是整个函数。
这段代码其实是这样的：

var arr[],i,j;
  for(i = 0;i<3;i++){
  arr.push(i);
  }
  for (j = 0;j<arr.length;j++){
  alert(arr[j]);}//输出3 3 3
