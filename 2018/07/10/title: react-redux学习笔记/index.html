react-redux学习笔记

Veröffentlicht am 2018-07-10 | in react . redux

主要概念

store

存储数据的地方，并且整个应用只能有一个store

action

表示用户触发的动作，意味着state要发生变化了

store.dispacth()

view发送action的唯一方式

createStore

createStore接受第二个参数，这个参数一般是服务器给出的，用于创建一个新的store对象

reducer

可以通过combineReducer将reducer合并成一个大的reducer

redux流程

（1）首先用户触发dispatch发送action
（2）其次调用reducer返回新的state
（3）然后state发生变化，subscribe进行监听
（4）listener通过getState获得最新状态

dva

数据改变
  • 数据发生改变时一般是用户交互触发或者是浏览器触发，通过dispatch触发action。
  • 如果是同步行为，通过触发Reducers改变state。
  • 如果是异步行为会先触发Effects然后流向Reducers最终改变state。
state

表示model的数据状态，可以是任何值

action

表示触发的动作，必须带有type属性以明确动作的具体行为，是改变state的唯一途径

dispatch

表示触发action的方式，在dva，connnect model里可以通过props访问到dispatch
